## ディレクトリ構成

### Web アプリ（React）側

```
./
│
├ .github/
|  └ workflows/                     # Github Action実行用のワークフロー
├ dist/                             # ビルド出力先（Git管理外）
├ node_modules/                     # ローカルNPMパッケージ（Git管理外）
├ public/                           # 画像などの静的リソース
├ src/
│  ├ components/                    # UIコンポーネント（アプリケーション全体で共有）
│  │   └
│  ├ features/                      # 機能
│  │   ├ api/                       # 機能毎のAPI関数
│  │   ├ components/                # 機能毎のUIコンポーネント
│  │   ├ hooks/                     # 機能毎のカスタムフック
│  │   └ types/                     # 機能毎の型定義
│  ├ pages/                         # 画面コンポーネント
│  |   └ App.tsx                    # React 基底コンポーネント（ルーティング、ライブラリ設定など）
│  ├ types/                         # 型定義（アプリケーション全体で共有）
│  ├ utils/                         # 共通部品（アプリケーション全体で共有）
│  ├ main.tsx                       # ルートJavascript（React コンポーネントを index.html へ注入）
│  └ vite-env.d.ts                  # Reactの環境変数の型定義ファイル
├ .gitignore                        # バージョン管理対象外指定ファイル
├ eslint.config.js                  # eslint設定ファイル。Viteプロジェクト生成時に自動生成。
├ index.html                        # ルートHTML（ブラウザに表示される html）
├ package-lock.json                 # プロジェクト情報記録ファイル
├ package.json                      # プロジェクト情報ファイル
├ tsconfig.app.json                 # Typescript設定ファイル（アプリケーション関連）
├ tsconfig.json                     # Typescript設定ファイル
├ tsconfig.node.json                # Typescript設定ファイル（Node.js関連）
└ README.md                         # 本プロジェクトの説明書
```
